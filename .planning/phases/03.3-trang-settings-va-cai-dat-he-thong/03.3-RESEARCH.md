# Phase 03.3: Trang Settings và Cài đặt hệ thống - Technical Research

## Objective
Research how to implement Phase 03.3: Trang Settings và Cài đặt hệ thống.
Answer: "What do I need to know to PLAN this phase well?"

## Findings: Architecture & Components

### 1. Panel Settings (AdminPanel.jsx)
- **Current State**: `AdminPanel.jsx` is just a placeholder showing "Tính năng đang phát triển".
- **Required Changes**:
  - Rename/Refactor to `SettingsPanel.jsx` (or keep `AdminPanel` but change its core).
  - It needs to list groups: Profile, Theme, Language, Notifications, Security (Password), World Editor.
  - When clicking "World Editor" (Chỉnh sửa không gian), it should toggle `isEditMode` state in `Docs.jsx`.

### 2. World Editor overlay and state (Docs.jsx)
- **Current Layout**: Isometric map is rendered in `Docs.jsx` with multiple zones. Models are rendered using `<model-viewer>`.
- **Edit Mode**:
  - We need a top-level state `isEditMode` in `Docs.jsx`.
  - When true, render an overlay to dim non-selected zones. Render grid on active zones.
  - Render Gizmos (handles) around the selected model to drag (x, y), scale, and rotate.
  - Also need a small floating form panel to input exact numerical values.
  - Preview & Confirm workflow: provide "Save Changes" and "Cancel" buttons floating on the screen when `isEditMode` is active.

### 3. Supabase Integration
- **`user_settings` table schema**: 
  - `id` (uuid, references auth.users)
  - `theme` (text)
  - `language` (text)
  - `notifications` (jsonb)
- **`world_config` table schema**:
  - `id` (uuid)
  - `user_id` (uuid, references auth.users)
  - `config` (jsonb) - `{ zones: { admin: { x, y, width, height } }, models: { lanterned_archive: { x, y, scale, rotation } } }`
- **AuthContext**: Need to support `updatePassword` and `uploadAvatar` via Supabase Storage.

### 4. Interactive Agent Navigation
- Instead of just navigating to the zone, we can add "clickable objects" in the Admin zone (which represents settings categories).
- Clicking them triggers `navigateToZone` (teleports agent to exact coordinates) AND opens the corresponding settings menu.

## Required Actions for Planning
1. **Supabase Setup**: Define the SQL to create `user_settings`, `world_config`, and the bucket for `avatars`.
2. **Context Extension**: Enhance `AuthContext` with avatar upload & password reset. Create a `WorldContext` or `SettingsContext` to manage settings and map positions state globally without prop drilling into `Docs.jsx`.
3. **Docs.jsx Refactor**:
   - Implement `isEditMode` overlay and interaction.
   - Replace static positions with values from `WorldContext`.
4. **AdminPanel Refactor**: Implement the actual settings UI, including the numeric panel for World Editing.
5. **Model Interactivity**: Add dragging logic for `<model-viewer>` containers when `isEditMode` is true.

## Validation Architecture
- E2E check: Toggle edit mode, move an object, save -> check Supabase row updated.
- Security: User can only read/update their own settings (RLS on Supabase).
- Auth: Theme and Profile successfully apply changes globally.
