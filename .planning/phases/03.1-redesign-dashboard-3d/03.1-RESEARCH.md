# Phase 03.1: Redesign Dashboard 3D - Technical Research

## Architecture Overview
The Dashboard (`Home.jsx`) currently uses a standard 2D layout. We will replace this with a 3D Isometric View (`isometric-world`) mimicking the mechanics built in `Docs.jsx`.

### Key Technical Challenges
1. **Full-screen Isometric Grid**: We need a `.isometric-container` that is `fixed inset-0` so the grid encompasses the entire screen.
2. **Pure CSS 3D Pedestals**: Creating 4 interactive 3D blocks (for Docs, Tools, Gallery, Admin) using pure CSS requires leveraging `transform-style: preserve-3d` and multiple faces (`front`, `back`, `left`, `right`, `top`). We will define standard CSS classes like `.cyber-pedestal` to render these cubes/cylinders on the `.isometric-world`.
3. **Agent Pathfinding/Movement**: The chibi agent (`Chibi Chef`) will need a localized React state `(x, y)` coordinate system on the grid instead of the current simple `agentPos`. 
4. **Triggering Navigation**: We must calculate the distance to the target when a pedestal is clicked, calculate animation duration, set a `setTimeout`, and invoke React Router's `navigate()` upon completion.

### Integration Points
- **React Router**: `useNavigate` hook for transferring to `/docs`, `/tools`, etc.
- **Model-Viewer**: Reusing Google's `<model-viewer>` web component with auto-toggling `src` (Walking vs Idle).

## Validation Architecture
- **Automated Tests**: N/A for this visual UI task.
- **Manual QA Requirements**:
  - Click on the `Docs` CSS Pedestal -> Verify character changes animation from Idle to Walking.
  - Verify character translates geometrically to the exact X/Y coordinate of the pedestal.
  - Verify that upon reaching the pedestal (after `n` seconds), the browser immediately redirects to `/docs`.
  - Verify CSS 3D structures render flawlessly without breaking the isometric perspective angle.
