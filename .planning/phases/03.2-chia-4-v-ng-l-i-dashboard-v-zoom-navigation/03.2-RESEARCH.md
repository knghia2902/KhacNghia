# Phase 03.2: Technical Research

## Architecture Assessment
- **Current State:** `Home.jsx` manages a `zone-grid` (2x2) using CSS grid with scroll wheel to route to other pages. `Docs.jsx` manages a complex set of cascading floating panels but does not have the 3D Scene integrated yet.
- **Target State (One-Map Canvas):** The application will merge navigation into a single Canvas view. Instead of switching pages (`/docs`, `/tools`), opening the App shows a giant map. The map holds multiple sub-areas ("islands/zones"). Floating UI panels (Docs Nav, Editor) overlay on top when a zone is selected.

## Technical Approach: Pan & Zoom Mechanism
- **Viewport Management:**
  1. The outer wrapper must act as a `viewport` (`overflow-hidden`, `w-full h-full`).
  2. Implement an intermediate `map-layer` that receives CSS transform styles for panning (X/Y) and zooming (Scale).
  3. Implement pointer event handlers (`onPointerDown`, `onPointerMove`, `onPointerUp`) mapped to React state (`panX`, `panY`, `isDragging`) to provide mouse drag functionality.
  4. Attach an `onWheel` event to adjust the `zoom` state within a bounded limit (e.g., 0.5x to 3.0x).
- **Isometric Grid:** Inside the map layer, the `.iso-floor` (or `.isometric-world`) holds the actual 3D rotation (`rotateX(60deg) rotateZ(-45deg)`). This enables placing standard 2D elements that suddenly project into the 3D plane.

## 3D Models & Dependencies
- **Technology:** `Home.jsx` relies on Google's `<model-viewer>` Web Component. We will adhere to this dependency as it simplifies loading and handling GLB files on the isometric plane. 
- **Assets Position:** Since the layout simulates `khacnghia.png`:
  - `Docs Zone`: Tủ Sách, giường, agent - đặt ở một khu vực (VD: bottom-right).
  - `Tools Zone`: Lò Rèn (`LoRen.glb`) - đặt ở khu vực khác (VD: top-left).
  - The map size needs to be large enough (e.g., `3000px x 3000px`) to distribute them.
- **Routing Integration:** Routing must ideally be replaced by `ActiveZone` state, but if physical path updates are required, deep linking should pan/zoom the camera directly into that zone on mount.
